GUIDE CONFIGURATION FIREBASE CONSOLE POUR GOOGLE AUTH
======================================================

PROBLÈME RÉSOLU : Cross-Origin-Opener-Policy et Google Auth

ÉTAPES À SUIVRE DANS FIREBASE CONSOLE :

1. ACCÉDER À FIREBASE CONSOLE
   - Aller sur https://console.firebase.google.com/
   - Sélectionner votre projet CineTrack

2. CONFIGURER LES DOMAINES AUTORISÉS
   - Dans le menu de gauche, cliquer sur "Authentication"
   - Aller dans l'onglet "Settings" (Paramètres)
   - Faire défiler jusqu'à "Authorized domains" (Domaines autorisés)

3. AJOUTER LES DOMAINES LOCAUX
   Cliquer sur "Add domain" et ajouter :
   - localhost
   - 127.0.0.1
   - localhost:4200
   - 127.0.0.1:4200
   - localhost:8080
   - 127.0.0.1:8080

4. CONFIGURER GOOGLE OAUTH
   - Aller dans l'onglet "Sign-in method"
   - Cliquer sur "Google" dans la liste des fournisseurs
   - Vérifier que le statut est "Enabled" (Activé)
   - Vérifier que l'email de support est configuré

5. VÉRIFIER LES CLÉS API
   - Aller dans "Project settings" (roue dentée)
   - Onglet "General"
   - Vérifier que les clés API sont configurées pour votre application web

SOLUTION TECHNIQUE IMPLÉMENTÉE :
================================

1. Méthode Fallback : popup → redirect
   - Si le popup échoue (Cross-Origin), redirection automatique
   - Gestion du retour de redirection dans LoginComponent

2. Configuration Google Provider
   - Ajout des scopes 'profile' et 'email'
   - Paramètre 'prompt: select_account' pour forcer la sélection

3. Gestion des erreurs améliorée
   - Détection des erreurs Cross-Origin
   - Basculement automatique vers la redirection

CODES D'ERREUR RÉSOLUS :
========================
- auth/popup-blocked
- auth/cancelled-popup-request  
- auth/internal-error
- Cross-Origin-Opener-Policy violations
